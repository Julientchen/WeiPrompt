## 🔴 高优先级问题 - 立即修复

### 问题 1: 单体组件过重 (App.vue 850行)
**严重程度**: 高
**问题位置**: `src/App.vue`

#### 问题描述
App.vue 单一文件包含了所有功能模块，严重影响可维护性和可扩展性。

#### 修复目标
将 App.vue 拆分为 6-8 个独立组件，每个组件职责单一。

#### 具体修复步骤
1. **创建组件目录结构**
   ```
   src/components/
   ├── HeaderBar.vue         # 顶部导航栏 (搜索、主题切换、新建按钮)
   ├── CategorySidebar.vue   # 分类侧边栏 (分类列表、标签云)
   ├── TemplateCard.vue      # 单个模板卡片 (列表展示用)
   ├── TemplateList.vue      # 模板列表页面
   ├── TemplateEditor.vue    # 模板编辑表单
   └── TemplatePreview.vue   # 模板预览与变量填写
   ```

2. **提取 HeaderBar 组件**
   - 移动 Logo、搜索框、主题切换、新建按钮
   - 保留事件 emit 供父组件处理

3. **提取 CategorySidebar 组件**
   - 移动分类列表、标签云
   - 保留选择分类的事件 emit

4. **提取 TemplateCard 组件**
   - 移动单个卡片的模板和逻辑
   - 支持点击、收藏事件 emit

5. **提取 TemplateEditor 组件**
   - 移动整个编辑表单
   - 支持保存、取消事件 emit

6. **提取 TemplatePreview 组件**
   - 移动变量填写和预览区域
   - 支持编辑、复制事件 emit

#### 验收标准
- [ ] App.vue 行数 < 150 行
- [ ] 每个组件职责单一，不超过 200 行
- [ ] 所有原有功能正常工作
- [ ] 主题切换、搜索、分类功能正常

### 问题 2: 死代码 - updateCategoryCounts 函数定义但从未调用

**问题位置**: `src/App.vue` (搜索 "updateCategoryCounts")

- [ ] 各个分类显示正确的计数

## 🟡 中优先级问题 - 近期修复

### 问题 3: 使用原生 alert() 进行错误提示
**严重程度**: 中
**问题位置**: `src/App.vue` (搜索 "alert")

#### 问题描述
两处使用原生 `alert()`，与整体设计风格不一致，用户体验差。

#### 具体位置
1. 保存模板验证: `alert('请输入模板标题')` / `alert('请输入模板内容')`
2. 复制失败: `alert('复制失败，请手动复制')`

#### 修复方案
替换为统一的 Toast 通知组件:
- 成功: 绿色背景 (参考复制成功提示)
- 错误: 红色背景
- 警告: 橙色背景

#### 验收标准
- [ ] 删除所有原生 alert() 调用
- [ ] 使用统一的 Toast 通知样式
- [ ] 通知 2 秒后自动消失
- [ ] 成功/错误状态有不同的视觉样式

### 问题 5: 缺少模板删除功能和确认机制
